# SEEK Quality Framework — Detailed Reference

## 01 Complex Made Simple

> "I apologise for such a long letter – I didn't have time to write a short one." — Mark Twain

**Core idea**: Make complex things simple through pragmatic, effective solutions. Each element serves a purpose. Leverage technology to reimagine flows.

### Tactics to Evaluate

**Reduce cognitive load**
- ✓ Navigation is intuitive and predictable
- ✓ Clear buttons, icons, and CTAs guide users
- ✓ Immediate feedback confirms actions
- ✓ Strategic colour draws attention to key interactions
- ✓ Consistent use of grids and spacing
- ✗ Users must remember information between screens
- ✗ Icons without labels or unclear meaning
- ✗ Too many competing visual elements

**Streamline decisions**
- ✓ Limited choices at each step (Hick's Law)
- ✓ Smart defaults reduce decisions needed
- ✓ Progressive disclosure hides complexity until needed
- ✗ Overwhelming option lists
- ✗ No clear recommended path
- ✗ Equal visual weight on all options

**Eliminate friction**
- ✓ Minimal steps to complete tasks
- ✓ Auto-save and smart persistence
- ✓ Pre-filled fields where possible
- ✗ Unnecessary confirmation steps
- ✗ Redundant data entry
- ✗ Dead ends with no recovery path

**Context-aware interfaces**
- ✓ Content adapts to user's journey stage
- ✓ Relevant actions surfaced at the right time
- ✓ Personalisation based on behaviour/preferences
- ✗ One-size-fits-all interfaces
- ✗ Static content regardless of context
- ✗ Features hidden in menus when frequently needed

## 02 Beautifully Crafted

> "Beauty is the purging of superfluities." — Michelangelo

**Core idea**: Harmonious blend of form, function, and emotion. Every detail is thoughtfully crafted with care and intentionality.

### Tactics to Evaluate

**Visual hierarchy**
- ✓ Size, colour, spacing guide attention
- ✓ Key information is immediately visible
- ✓ Consistent typography scale
- ✗ Everything competing for attention
- ✗ Important content buried or small
- ✗ Inconsistent heading sizes

**Responsive design**
- ✓ Layouts adapt gracefully across breakpoints
- ✓ Touch targets appropriate for device
- ✓ Content reflows logically
- ✗ Horizontal scrolling on mobile
- ✗ Tiny tap targets on touch devices
- ✗ Desktop layout crammed into mobile

**Functional visuals**
- ✓ Visual elements enhance understanding
- ✓ Icons support (not replace) text
- ✓ Illustrations serve a purpose
- ✗ Decorative elements that add noise
- ✗ Stock imagery that doesn't relate to content
- ✗ Over-styled UI that obscures function

**Balanced spacing**
- ✓ Negative space highlights key elements
- ✓ Content density appropriate for audience
- ✓ Consistent spacing rhythm
- ✗ Cramped layouts with no breathing room
- ✗ Excessive whitespace that wastes screen real estate
- ✗ Inconsistent padding/margins

**Micro-interactions**
- ✓ Subtle animations provide feedback
- ✓ Transitions feel smooth and purposeful
- ✓ Loading states are designed, not generic
- ✗ Jarring state changes with no transition
- ✗ Over-animated interfaces that feel slow
- ✗ Missing feedback on user actions

## 03 Purposefully Innovative

> "Innovation distinguishes between a leader and a follower." — Steve Jobs

**Core idea**: Introduce novel ideas while maintaining usability. Balance innovation with accessibility. Innovation must align with SEEK brand.

### Tactics to Evaluate

**Strive for originality**
- ✓ Distinctive design elements that differentiate SEEK
- ✓ Fresh approaches to common patterns
- ✓ Brand-aligned innovation
- ✗ Generic templates indistinguishable from competitors
- ✗ Innovation for its own sake
- ✗ Off-brand experimental design

**Leverage cutting-edge technology**
- ✓ AI/ML features that enhance the experience
- ✓ Modern web capabilities used appropriately
- ✓ Technology serves user needs
- ✗ Tech gimmicks without user value
- ✗ Bleeding-edge features that exclude users
- ✗ Legacy patterns when better solutions exist

**Balance innovation with usability**
- ✓ New patterns are learnable
- ✓ Familiar conventions respected where helpful
- ✓ Accessibility maintained in innovative features
- ✗ Clever designs that confuse users
- ✗ Breaking established mental models without good reason
- ✗ Innovation that excludes users with disabilities

**Use trends wisely**
- ✓ Trends adopted selectively with purpose
- ✓ SEEK's unique identity maintained
- ✓ Timeless over trendy for core patterns
- ✗ Trend-chasing that dates quickly
- ✗ Copying competitors without adaptation
- ✗ Inconsistent adoption of trends across product

## 04 Cohesive at Every Touchpoint

> "Consistency is the foundation of virtue." — Francis de Sales

**Core idea**: Leverage existing design systems and guides for familiar, intuitive design. 80% Braid, 20% custom patterns.

### Tactics to Evaluate

**Design system compliance**
- ✓ 80%+ Braid components used
- ✓ Custom patterns documented and justified
- ✓ Consistent component usage across product
- ✗ Rebuilding what Braid provides
- ✗ Inconsistent component variations
- ✗ Undocumented custom patterns

**Standardised templates**
- ✓ Common page types follow templates
- ✓ Predictable content placement
- ✓ Reusable patterns for similar content
- ✗ Every page designed from scratch
- ✗ Inconsistent information architecture
- ✗ Users must relearn navigation per section

**Cross-device consistency**
- ✓ Same mental model across platforms
- ✓ Features available on all relevant platforms
- ✓ Consistent terminology and labelling
- ✗ Different flows per platform without reason
- ✗ Features missing on mobile/desktop
- ✗ Naming varies across touchpoints

**Writing style guide**
- ✓ Consistent voice and tone
- ✓ Terminology matches SEEK standards
- ✓ Clear, concise copy
- ✗ Inconsistent capitalisation or formatting
- ✗ Jargon or unclear language
- ✗ Mixed voice (formal in some places, casual in others)

**End-to-end approach**
- ✓ Upstream/downstream impacts considered
- ✓ User journey spans designed together
- ✓ Handoffs between flows are smooth
- ✗ Features designed in isolation
- ✗ Jarring transitions between product areas
- ✗ Inconsistent data or state across flows

## 05 Embrace Diversity

> "The best designs are those that are accessible to everyone, regardless of age, ability, or culture." — Don Norman

**Core idea**: Understand and embrace cultural and societal diversity across APAC. Accessible to all.

### Tactics to Evaluate

**Accessibility (AA compliance)**
- ✓ Colour contrast meets WCAG AA (4.5:1 text, 3:1 UI)
- ✓ Supports dynamic text sizing
- ✓ Works with screen readers
- ✓ Keyboard navigable
- ✓ Respects reduced motion preferences
- ✗ Contrast failures
- ✗ Fixed text sizes
- ✗ Missing alt text or ARIA labels

**Representation**
- ✓ Diverse people in illustrations/photos
- ✓ Inclusive language
- ✓ Avoids stereotypes
- ✗ Homogeneous imagery
- ✗ Gendered or exclusionary language
- ✗ Cultural stereotypes

**Cultural sensitivity**
- ✓ Colours appropriate for market
- ✓ Imagery culturally relevant
- ✓ Local conventions respected (date formats, etc.)
- ✗ Colours with negative cultural associations
- ✗ Imagery that doesn't resonate locally
- ✗ Assumptions based on Western norms

**Readability**
- ✓ Headers and sections break up content
- ✓ Scannable layouts
- ✓ Simple, clear language
- ✗ Walls of text
- ✗ Complex sentences or jargon
- ✗ Poor information hierarchy

---

## Quality Framework Priorities

When qualities tension, prioritise:

1. **Embrace Diversity** — Accessibility is non-negotiable
2. **Complex Made Simple** — Usability over aesthetics
3. **Cohesive at Every Touchpoint** — Consistency builds trust
4. **Beautifully Crafted** — Polish matters but not at cost of function
5. **Purposefully Innovative** — Innovation is aspirational, not mandatory
